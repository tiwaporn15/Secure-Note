# SecureNote

A full-stack secure notes app — React frontend + Node.js/Express backend with PocketHost cloud persistence.

## Project Structure

```
/secure-note-app
├── /backend
│   ├── .env              ← Secrets (NOT committed to Git)
│   ├── .env.example      ← Safe template
│   ├── .gitignore
│   ├── server.js         ← Express API
│   └── package.json
├── /frontend
│   ├── /src
│   │   ├── /pages        ← LoginPage, NotesPage
│   │   ├── /components   ← NoteCard, ComposePanel
│   │   ├── App.jsx
│   │   ├── main.jsx
│   │   └── index.css
│   ├── index.html
│   ├── vite.config.js
│   └── package.json
├── REPORT.md
└── README.md
```

## Prerequisites

- Node.js v18+
- npm

## Setup

### 1. Backend

```bash
cd backend
cp .env.example .env
# Edit .env and set PORT, SECRET_TOKEN, POCKETHOST_TOKEN
npm install
npm start
# → http://localhost:3001
```

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
# → http://localhost:5173
```

Open `http://localhost:5173` in your browser. Sign in with the `SECRET_TOKEN` from `backend/.env`.

## API

| Method | Endpoint          | Auth | Description       |
|--------|-------------------|------|-------------------|
| GET    | `/api/notes`      | No   | Get all notes     |
| POST   | `/api/notes`      | Yes  | Create note       |
| DELETE | `/api/notes/:id`  | Yes  | Delete note       |

Auth = `Authorization: <SECRET_TOKEN>` header.

## Security

- `.env` is git-ignored — secrets never enter the repository
- `SECRET_TOKEN` is never in frontend source code
- Always use HTTPS in production to encrypt tokens in transit
